AstraNotes UML Activity Diagram

Main user and system flows, expanded to cover biometric fallback policy, settings validation, plugin manifest checks, secure search behavior, and key rotation recovery.

Notes

  • All flows are derived from the requirements and architecture documentation.
  • Search flow distinguishes locked results from unlocked in-memory secure-title matching.
  • Auto-lock explicitly persists secure drafts before clearing keys and secure-title cache.
  • Import and key-rotation flows now show conflict resolution, rollback behavior, and interrupted-rotation recovery.
  • Unlock flow now models biometric fail-count threshold fallback and unavailable-hardware fallback.
  • Plugin flow now models manifest validation and explicit text-transform action execution.
  • Settings flow now models validation and commit of lock timeout, telemetry, and plugin global toggle.
  • Secure-note open flow now includes authenticated-decryption failure with preserved stored record.
