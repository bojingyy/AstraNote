AstraNotes UML Object Diagram

Runtime snapshot: unlocked session with an active secure note, secure title cache loaded, and plugin-enabled editing flow.

Snapshot assumptions

  • This object diagram captures one concrete runtime moment with an unlocked session and a selected secure note.
  • Object names follow instance notation objectName:ClassName to distinguish runtime objects from class-level design.
  • Secure note title visibility is represented by secureCache:SecureTitleCache, which remains memory-only and non-persistent.
  • Repository objects are connected to db:DatabaseProvider through dashed transaction links.
  • Plugin action path is shown as pluginSvc:PluginService writing outcome through noteSvc:NoteService and metadata repositories.
