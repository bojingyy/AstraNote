AstraNotes UML Object Diagram

Representative runtime snapshot derived from the current codebase: a later-launch session is unlocked, ContentView has routed to NotesWorkspaceView, a secure note is open, one attachment is loaded, and search/trash/settings data are available through the live services.

What Changed In This Revision

  • The diagram now reflects the real runtime graph from the Swift code: AppEnvironment as the composition root, ContentView routing into NotesWorkspaceView, actor-based services, repository actors, DatabaseProvider, and the platform objects actually instantiated by the app.
  • The old stale objects were removed: AppState, TrashView, NotificationService, SecureNotePolicyService, SecureTitleCache, Note/TrashItem/SubjectGroup placeholders that do not exist as runtime types, plus non-existent settings like lockTimeoutSec and telemetryOptIn.
  • The snapshot is intentionally a later-launch unlocked session, because the actual app only shows UnlockView during .firstLaunchSetup; both .locked and .unlocked route to NotesWorkspaceView.
  • Search is shown through NoteSearchResult values sourced from persisted secureTitleAlias; there is no decrypted secure-title cache in the implementation.
  • Arrow routing was redrawn into cleaner corridors so labels stay readable and connectors avoid crossing card text.
  • Fixed in this revision: The imageAttachment:Attachment object card was missing the createdAt field, which is a required field on the Attachment struct in CoreModels.swift. The field has been added at the bottom of the card, consistent with the same fix applied to the class diagram.
