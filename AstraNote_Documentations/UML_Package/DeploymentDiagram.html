AstraNotes UML Deployment Diagram

Deployment view aligned with the implemented local-first macOS app: one desktop process, in-process JSON-backed persistence, App Support attachment storage, Keychain plus LocalAuthentication for biometrics, and user-selected local files for backup/import and plugin installation. The layout separates physical zones so connectors remain clear and non-overlapping.

Alignment Notes

  • This version reflects the implemented storage model: DatabaseProvider persists one JSON snapshot at ~/Library/Application Support/AstraNotes/database-state.json. The previous SQLite node was removed because it does not exist in the code.
  • Attachment bytes are stored separately in the app's Attachments directory, while attachment metadata lives in the JSON snapshot. Plugin bundle bytes are persisted inside the snapshot model, not as a separate installed-plugin file store.
  • The biometric path is shown as a combination of LocalAuthentication and a Keychain item with kSecAttrAccessibleWhenUnlockedThisDeviceOnly. In-memory note keys are intentionally not persisted anywhere on disk.
  • Backup export/import, image attach, and plugin install all use user-selected local files via macOS file panels. There is no cloud service, remote API, telemetry sink, secure-note expiry notification path, or network plugin marketplace in this deployment.
  • The platform-event stream is modeled as an app-side integration concept with a subscription present in the coordinator, but the current build does not wire an OS lifecycle/power producer into it yet, so this deployment diagram avoids overstating that path as a fully deployed callback pipeline.
