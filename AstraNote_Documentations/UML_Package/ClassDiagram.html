AstraNotes UML Class Diagram

Architecture-level class view derived from source code. All types, method names, and fields reflect the actual implementation.

Key architecture facts reflected in this diagram

  • AstraUI: NotesWorkspaceView is a single ~1,300-line composition root. There are no separate WorkspaceTopBar, SubjectSidebarPane, NoteCollectionPane, or NoteEditorPane types — all pane UI is inline.
  • Session states: AppCoordinator.SessionState has exactly three cases: firstLaunchSetup, locked, unlocked. There is no expiration sweep, inactivity timeout, or lock-timeout setting.
  • KeyManager: Key material is never persisted or restored from disk. Lockout doubles from 30 s to a maximum of 3,600 s. changePassphrase re-encrypts all secure notes atomically with a recoverable pendingCredentialRotation marker.
  • NoteSearchService: clearSecureCacheOnLock() and secureCacheCount() are intentional no-ops — there is no decrypted-title cache.
  • AppSettings: Contains exactly two Bool fields: pluginsEnabled and biometricUnlockEnabled. No lockTimeoutSeconds, telemetryOptIn, or any other field.
  • EncryptionService: Exposes only encrypt and decrypt. No reencrypt method.
  • Removed from diagram (non-existent in code): AppState, SecureNotePolicyService, StorageProtection, NotificationService, SecureTitleCache, separate pane view types, expiresAtUtc, lockTimeoutSeconds, telemetryOptIn, oldKey on KeyMaterial, supportedAppVersion on PluginManifest.
