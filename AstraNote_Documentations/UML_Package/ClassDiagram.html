AstraNotes UML Class Diagram

Complete architecture-level class view from Architecture.md, including UI, Core, Data, Platform, Models, and in-memory session cache.

Relationship notes

  • UI classes depend only on core services and AppCoordinator; they do not call repositories or encryption directly.
  • NotesWorkspaceView composes WorkspaceTopBar, SubjectSidebarPane, NoteCollectionPane, and NoteEditorPane.
  • NoteService delegates secure data transformation to EncryptionService and persistence to NoteRepository and AttachmentRepository.
  • SecureNotePolicyService collaborates with ProtectedTrashService, NotificationService, and TimeProvider for expiration flow.
  • SettingsService drives passphrase rotation through KeyManager and updates settings through SettingsRepository.
  • PluginService validates manifests and stores plugin runtime state through PluginMetadataRepository and PluginBundleRepository.
  • All repositories depend on DatabaseProvider transaction boundaries.
