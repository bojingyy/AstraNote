1) Chosen development direction
- Programming language and framework: Swift 5.7+ with SwiftUI.
- Platform decision: desktop app (macOS 12+), local-first.
- Why this direction: it matches the project requirements for biometric unlock, local protected storage, secure note handling, and the architecture already centered on macOS platform services.

2) Project structure and build/run environment
- Structure follows layered modules:
  - AstraUI: SwiftUI screens and pane composition.
  - AstraCore: coordinator and domain services (notes, secure policy, search, trash, plugins).
  - AstraData: SQLite/GRDB repositories and transaction boundaries.
  - AstraPlatform: LocalAuthentication, notifications, time provider, storage protection.
- Build/run environment:
  - Xcode project workflow (`AstraNotes.xcodeproj`).
  - Recommended toolchain: Xcode 14+, Swift 5.7+, macOS 12+.
  - Command-line test path: `xcodebuild test -scheme AstraNotes -destination 'platform=macOS'`.

3) UI shell summary
- Main shell is a tri-pane notes workspace:
  - Left pane: foldable subject sidebar with subject groups and "All Notes".
  - Middle pane: note list/cards filtered by subject and title search.
  - Right pane: note editor with secure toggle and expiration date/time controls in the top-right toolbar.
- Top-level navigation/surfaces include:
  - Workspace top bar (new note, search, voice capture, secure indicators).
  - Profile/session surface through unlock/biometric flow.
  - Settings view (lock timeout, telemetry opt-in, plugin global toggle).
  - Trash view (restore/permanent delete with lock badge handling for secure notes).

4) First functionality slices implemented or drafted
- Slice A: Secure note lifecycle + expiration + protected trash.
  - Drafted behavior: opt-in secure mode per note, mandatory expiration date/time, ciphertext-at-rest save path, expiration checks, move-to-trash and restore/permanent-delete logic.
- Slice B: Title search with lock-aware secure handling.
  - Drafted behavior: normal notes searched from storage, secure titles searchable only while unlocked from in-memory decrypted cache, cache cleared on lock.
