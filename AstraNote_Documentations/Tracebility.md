| FR1.3 | After a successful passphrase unlock, the user may optionally enable biometric unlock for future sessions. | `AppCoordinator`, `KeyManager`, and `LocalAuthService` relationships indicate session and biometric orchestration. | FR2.5 | Deleting a normal note shall move it and its attachments to protected trash in a single ACID transaction; if the move fails, the note record shall remain intact. | `NoteService.deleteNote()`, `ProtectedTrashService.moveToTrash()`, `NoteRepository`, `ProtectedTrashRepository`, and `DatabaseProvider.transaction()` provide the structural transaction path. | Activity includes `Delete note -> Move note to trash`; trash flows show restore/permanent delete behavior. | Deployment includes shared local SQLite persistence for notes/attachments/trash, enabling transactional semantics at storage layer. | Partially Traced | UML does not explicitly show the single transaction boundary across note plus attachments, nor explicit rollback branch when move fails. |
| FR3.6 | If decryption of a secure note fails due to invalid ciphertext or authentication data, the stored record shall be preserved and a user-visible error shall be shown. | `EncryptionService.decrypt(...)` and `NoteService` secure-note handling indicate decrypt path; object model links note operations to encryption service. | No explicit use case for decrypt failure handling. | FR6.3 | Audio exceeding 10 minutes or 50 MB shall be rejected before storage with a message stating the applicable limit. | UI and note-service structure supports voice capture entry point (`NotesWorkspaceView` to core services) but no dedicated recorder validator class is modeled. | Activity explicitly models `Record audio -> Under 10 min and 50 MB? -> RejectAudio`. | NFR3.2 | Decrypted secure material, including secure note titles used for search, shall be held in memory only for the active unlocked session and cleared on lock. | `NoteSearchService.clearSecureCache()`, `KeyManager.clearKeys()`, and `AppCoordinator.lock()` directly model in-memory secure-data clearing on lock.
